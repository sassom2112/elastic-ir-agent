Elastic IR Agent

An autonomous incident response agent that turns raw Windows attack telemetry into a structured, evidence-backed IR report — no analyst in the loop. Gemini 2.5 Flash reasons over 73,909 real attack events indexed in Elastic, calls ES|QL detection tools via MCP, and runs an independent Forensic Auditor that challenges every MITRE ATT&CK claim with fresh evidence.

Security Model

Bad actions are architecturally impossible — not instructed away.

ControlImplementation
Session isolationsearch_memory hard-scoped to session_id at the Elasticsearch filter — model cannot query across investigations
Input validationtime_window, host_name, threshold validated before any ES call — regex allowlist + DoS cap (100y max)
Write allowlistwrite_memory checks target index against hardcoded allowlist — misconfigured ELASTIC_INDEX_MEMORY is blocked at write time
Content sanitizationwrite_memory strips control characters, caps at 10,000 chars — blocks indirect prompt injection via poisoned retrieval
Split API keysELASTIC_API_KEY_READ (ES|QL) and ELASTIC_API_KEY_WRITE (memory index only) — separate scoped keys
Structural labelingTool results wrapped with _provenance metadata — model cannot mistake retrieved data for instructions
Audit logEvery tool call atomically appended to audit_log.jsonl via os.open/os.write — blocked calls log the rejection reason
